# N수 1년 루프 Plan

## 목표

수능 결과 화면에서 재수 또는 N수를 선택했을 때 즉시 수능 5과목 전투로 들어가지 않고, 다시 1년 과정의 월별 전투를 거친 뒤 수능 5과목 전투를 다시 치르고 결과를 산출하도록 수정한다.

## 현재 분석

- 결과 화면의 `N수 선택`은 `Hr(...)` 흐름에서 `gradeId`를 `REPEATER`로 변경하고 `retakeCount`를 증가시킨다.
- `REPEATER` 학년 데이터는 `phase: "repeater"`이며, `grade_visuals.json`에는 12개월 월별 전투용 비주얼이 이미 준비되어 있다.
- 전투 생성 함수 `br(...)`가 `H3`와 `phase === "repeater"`를 같은 조건으로 묶어 즉시 `yr(...)` 수능 5과목 전투를 생성하고 있었다.
- 전투 완료 함수 `Dr(...)`는 `battle.kind === "suneung"`일 때만 결과/선택 화면으로 들어가고, 그 외 전투는 다음 학년 진행으로 처리한다.
- 결과 산출 함수 `Er(...)`는 `battle.kind === "suneung"`일 때만 수능 시험 커브와 이름을 사용한다.

## 문제점

- N수를 선택하자마자 수능 전투가 발생해 1년 동안 새 변수나 성장 기회를 만들 수 없다.
- `REPEATER`의 12개월 전투 자산이 존재하지만 실제 흐름에서 사용되지 않는다.
- 단순히 `REPEATER`를 일반 학년 전투로 바꾸면 완료 후 결과 화면으로 돌아오지 않고 다음 진행 로직으로 빠질 위험이 있다.

## 구현 방향

1. 전투 생성
   - `H3`만 즉시 수능 5과목 전투를 생성한다.
   - `REPEATER`는 일반 학년처럼 12개월 전투를 생성한다.

2. 전투 완료
   - `REPEATER` 12개월 전투 완료 시 학년을 넘기지 않고 `gradeId`를 유지한다.
   - `REPEATER` 12개월 전투 완료 직후 `suneung` 전투를 새로 생성한다.
   - `battle.kind === "suneung"` 완료 시에만 결과/선택 화면으로 들어간다.

3. 결과 산출
   - `REPEATER` 12개월 전투는 학년 평가 기록으로 남긴다.
   - `REPEATER` 이후 수능 5과목 전투 결과는 수능 시험 커브와 `수능시험` 이름을 사용한다.
   - 직업 후보와 대학 합격권 산출은 기존 수능 결과 로직을 그대로 재사용한다.

4. 검증
   - N수 선택 직후 `REPEATER` 상태와 12개월 전투 카드 12개를 확인한다.
   - 즉시 수능 5과목 카드가 나오지 않는지 확인한다.
   - 12개월 전투 완료 후 수능 5과목 카드 5개가 생성되는지 확인한다.
   - 수능 5과목 전투 완료 후 결과/선택 화면으로 돌아오고 직업 후보가 다시 산출되는지 확인한다.
   - 모바일 smoke와 함께 기본 검증에 포함한다.

## 사용 도구와 MCP

- Node 스크립트: 스냅샷 번들 패치와 검증 자동화
- Playwright: 모바일 화면에서 N수 선택 및 전투 완료 흐름 smoke
- Godot/Unity/Unreal/UMG MCP: 이번 웹/APK 방향 작업에는 사용하지 않는다.

## 완료 기준

- N수 선택 직후 화면이 `N수 과정 · 12개월 전투`로 표시된다. 완료
- 월별 전투 카드가 12개 생성되고 수능 5과목 카드는 즉시 생성되지 않는다. 완료
- N수 1년 전투 완료 후 수능 5과목 전투가 다시 생성된다. 완료
- N수 수능 전투 완료 후 다시 결과 대기/직업 선택 흐름으로 돌아온다. 완료
- `npm run verify:mobile`에서 N수 smoke까지 함께 통과한다. 완료
