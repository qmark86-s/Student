# 부동산 상세 지도 확대/축소 조작 계획

## 요약
- 부동산 지역 상세 화면의 기본 카메라를 기존 확대 초점 상태에서 전체 지역이 보이는 축소 상태로 변경한다.
- 모바일은 두 손가락 핀치 확대/축소와 드래그 이동을 지원한다.
- PC는 상세 지도 viewport가 포커스된 상태에서만 마우스 휠로 확대/축소한다.

## 구현 항목
1. 상세 지도 카메라
   - 내부 상세 맵은 기존 `200%` 크기를 유지한다.
   - 기본/최소 배율은 `0.5`로 두어 한 화면에 전체 맵이 들어오게 한다.
   - 최대 배율은 기존 상세 화면 수준인 `1`로 제한한다.
   - 배율에 따라 팬 가능한 최소/최대 좌표를 다시 계산해 빈 공간이 보이지 않게 한다.

2. 입력
   - 상세 viewport는 `tabIndex=0`으로 포커스 가능하게 만든다.
   - PC 휠 확대/축소는 상세 viewport가 포커스된 상태에서만 처리한다.
   - 포인터 입력은 1개일 때 드래그 이동, 2개 이상일 때 핀치 확대/축소와 중심 이동을 처리한다.
   - 지도 클릭/터치 시작 시 상세 viewport에 포커스를 준다.

3. 스타일
   - 상세 맵 변환 기준점을 좌상단으로 고정한다.
   - 상세 viewport에 focus-visible 표시를 추가한다.
   - 지도 영역에서 브라우저 스크롤/선택이 섞이지 않도록 touch/overscroll/user-select 설정을 보강한다.

## 검증
- `npm run react:build`
- `npm run react:real-estate-smoke`

## 완료 기준
- 상세 화면 진입 직후 `data-zoom="0.50"` 상태로 시작한다.
- 상세 viewport 포커스 후 PC 휠로 `data-zoom`이 증가한다.
- 확대된 상태에서 드래그하면 `data-pan-x`, `data-pan-y`가 변경된다.
- 기존 부동산 overview, 상세 baked PNG, 구매/개발도 흐름이 유지된다.
