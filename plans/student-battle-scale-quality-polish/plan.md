# 학생 전투 스케일 품질 폴리싱 계획

## 배경

학생 탭 전투 화면에서 학습 도우미가 주인공 대비 너무 작고 눌려 보이며, 일반 몬스터도 작거나 투명하고 찢어진 것처럼 보이는 사례가 확인됐다. 화면상 아트 품질을 직접 해치는 문제이므로 표시 크기, 클리핑, 검증 기준을 함께 수정한다.

## 목표

- 학습 도우미는 주인공의 약 0.7~0.8배로 보이게 한다.
- 일반 몬스터는 기존 모바일 표시 크기 대비 약 2.5배 수준으로 키운다.
- 보스/수능 몬스터는 일반 몬스터보다 작아 보이지 않게 별도 표시 크기를 유지한다.
- 크기 조정값은 `data/battle_road_config.json`에서 관리하고 한글 도움말을 둔다.
- 실제 화면 검증에서 학습 도우미와 일반 몬스터의 최소 크기, 클리핑, 투명/누끼 이상을 잡는다.

## 작업 항목

1. 기존 학습 도우미, 일반 몬스터, 보스 몬스터 표시 크기와 슬롯 위치를 확인한다.
2. `battle_road_config.json`에 표시 크기 파라미터를 추가한다.
3. `tools/build-visual-assets.mjs`가 표시 크기 파라미터를 CSS 변수/규칙으로 생성하게 한다.
4. 몬스터 슬롯 위치를 큰 사이즈에 맞게 재배치한다.
5. `tools/visual-asset-smoke.mjs`에 화면상 최소 크기와 클리핑 검사를 추가한다.
6. `npm run build:web`, `npm run visual:smoke`, `npm run asset:factory:qa`, `npm run verify:mobile`로 검증한다.
7. 캡처 이미지를 직접 보고 아트 품질이 낮은 케이스가 남으면 추가 수정한다.

## 완료 기준

- 학생 전투 캡처에서 학습 도우미가 동료처럼 읽힌다.
- 일반 몬스터가 화면에서 작고 흐린 점처럼 보이지 않는다.
- 학생/도우미/몬스터가 화면 밖으로 잘리거나 서로 심하게 겹치지 않는다.
- 모든 자동 검증이 실패 없이 통과한다.
