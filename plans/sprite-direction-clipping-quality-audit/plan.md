# 스프라이트 방향/클리핑/품질 전수 검수 계획

## 목표

- 학생/동료/학습도우미는 오른쪽(`->`), 원정대 몬스터는 왼쪽(`<-`) 방향 기준을 유지한다.
- 원정대 탭에서 캐릭터와 몬스터가 화면 또는 셀 경계에 잘려 보이지 않게 한다.
- 원화, 정규화 프레임, 스프라이트 아틀라스, 실제 화면을 모두 비교해 낮은 품질 자산을 방치하지 않는다.
- 브라우저 애니메이션 중 아틀라스 중간 좌표가 샘플링되어 옆 프레임이 섞이는 문제를 자동 검증으로 잡는다.
- 160px 셀 안에서 머리와 발이 붙어 보이지 않도록 상하 여백을 8px 이상 확보한다.

## 작업 순서

1. manifest와 축 리포트에서 방향, 프레임 수, 기준선, 중심축을 확인한다.
2. 원정대 실제 화면을 Playwright로 캡처하고 DOM/가상요소 위치를 덤프해 클리핑 원인을 분리한다.
3. 원정대/학습도우미 스프라이트 키프레임이 선언된 프레임 위치만 사용하도록 보정한다.
4. 스프라이트 기준선을 `baselineY=151`로 조정하고 상하 여백 감사를 추가한다.
5. smoke 테스트에 프레임 스냅 검사를 추가해 중간 `background-position` 보간을 실패로 처리한다.
6. `asset:factory:review`, `visual:smoke`, `verify:mobile`로 전수 검증한다.

## 완료 기준

- `SPRITE_INTEGRITY_OK`가 통과한다.
- `visual:smoke`에서 원정대 동료/몬스터 클리핑 0, 수평 오버플로 0, 프레임 스냅 오차 0.05% 이하를 만족한다.
- `asset:integrity`에서 전체 프레임 상단/하단 여백 8px 이상을 만족한다.
- 실제 원정대 스크린샷에서 캐릭터나 몬스터가 반쯤 잘린 것처럼 보이지 않는다.
- 구현 설명서에 원인, 수정 방식, 검증 명령을 남긴다.
