# 학생/원정대 UI 밀도 개선 계획

## 목표
- 원정대 탭과 학생 탭 전체를 대상으로 긴 텍스트, 중복 정보, 항상 펼쳐져 있는 세부 정보를 줄인다.
- 핵심 상태와 즉시 행동은 접힌 상태에서도 보이게 하고, 설명/세부/고급 정보는 사용자가 요청할 때 펼친다.
- 기존 게임 규칙, 저장 구조, 데이터 생성 방식은 변경하지 않는다.

## 조사 기준
- NN/g의 progressive disclosure 기준처럼 초기 화면에는 핵심 옵션만 보여주고, 보조/고급 정보는 요청 시 노출한다.
- Carbon Design System의 accordion 사용 기준처럼 세로 공간이 부족한 모바일에서는 큰 콘텐츠를 접힘 영역으로 나눌 수 있다.
- 단, 사용자가 대부분의 내용을 항상 비교해야 하는 영역에는 접힘을 남용하지 않는다. 핵심 수치와 액션은 요약 행에 유지한다.

## 전수검사 결과
- 원정대
  - `의뢰`: 이미 요약 카드 + 세부 접힘 구조로 정리됨.
  - `성장`: 헤더와 카드에 “투자 가능/보유/필요” 문구가 반복된다. 핵심은 EXP, 레벨, 증가 전투력, 비용이다.
  - `파티`: 후보 카드의 목적은 편성 여부와 전투력 비교다. 세부 전투 스탯 반복은 줄인 상태이며, 슬롯 문구도 더 짧게 만들 수 있다.
  - `대원 관리`: 합성 후보가 없을 때 매번 빈 안내가 노출되어 관리 목록 스캔을 방해한다.
  - `기록`: 마지막 전투 이벤트가 항상 펼쳐져 있어 기록 탭이 길어진다.
- 학생
  - `성장`: 자동 투자 비율은 자주 바꾸는 설정이 아니므로 기본 접힘으로 이동할 수 있다.
  - `시험`: 현재 전투/진행 정보 중심이라 접힘 대상이 적다.
  - `직장`: 요약 카드와 “졸업생 직업 수입” 카드가 수입/슬롯 정보를 반복한다.
  - `교육`: 9개 교육 카드가 “생산/다음” 문구를 반복한다. 수치 중심으로 축약할 수 있다.
  - `결과`: 직업 목록은 이전 작업에서 정렬/스크롤/텍스트 축약을 적용했다.
  - `도감`: 직업 카드 62개가 과목 분배 가이드를 항상 펼쳐 보여 가장 큰 밀도 문제가 있다.

## 구현 범위
- 공통 `CompactDisclosure` 컴포넌트를 추가한다.
  - 제목, 보조 메타, `세부/접기` 버튼, 회전 chevron을 제공한다.
  - 카드 내부 또는 패널 단위 접힘에 모두 사용할 수 있게 CSS를 공통화한다.
- 원정대
  - 성장 헤더/빈 상태/슬롯 문구를 짧게 정리한다.
  - 대원 관리에서 합성 후보가 없을 때 빈 카드를 숨긴다.
  - 기록 탭의 마지막 전투 이벤트를 접힘 상세로 이동한다.
- 학생
  - 성장 탭의 자동 투자 비율 패널을 기본 접힘으로 바꾼다.
  - 요약 metric 라벨을 짧게 조정한다.
  - 직장 탭의 중복 수입 카드를 제거하고 요약 라벨을 축약한다.
  - 교육 카드 효과 문구를 `현재 → 다음` 수치 중심으로 줄인다.
  - 도감의 발견 직업 요약 중복 그리드를 제거하고, 직업 카드의 과목 분배 가이드는 카드별 `세부` 접힘으로 이동한다.
